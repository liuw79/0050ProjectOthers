## 为什么要做圈子？

2025年，我们完成了从"个人作战"到"团队协作"的转变。但团队还缺少正式的授权机制——大家背靠背信任了，却没有被组织正式"允许"自主决策。

同时，企服业务验证成功，业务复杂性增加。原来的部门中心结构开始出现：**响应慢、接口乱、闭环难**。

2026年，我们用"圈子"来解决这些问题。

---

## 什么是圈子？

**圈子 = 价值作战单元**

每个圈子要回答两个问题：
1. 我的客户是谁？
2. 我给他创造什么价值？

```
┌─────────────────────────────────────┐
│              高维组织               │
│  ┌───────┐  ┌───────┐  ┌───────┐   │
│  │ 圈子A │  │ 圈子B │  │ 圈子C │   │
│  │(业务) │  │(产品) │  │(运营) │   │
│  └───────┘  └───────┘  └───────┘   │
│         ↘      ↓      ↙            │
│          共同面向客户               │
└─────────────────────────────────────┘
```

**三大原则：**

| 原则 | 含义 |
|-----|-----|
| **高内聚** | 圈子内完成大部分工作，减少跨圈协调 |
| **价值导向** | 一切导向客户价值，内部圈子也要服务业务圈子 |
| **自主决策** | 权力下放，圈子内的事圈子自己定 |

**核心角色：**
- **圈长**：定目标、调资源、决定人员进出
- **接口人**：对外沟通协调（没有圈长时）
- **成员**：1-3个角色，可跨圈，责任清晰

**你现在要想的：**
- 我能为哪个圈子创造什么独特贡献？
- 我要不要报名当圈长？
- 我想加入哪个圈子？

## 一句话总结

> **圈子关注价值创造，社区关注伙伴成长。**
>
> 我们不是在搞"圈子文化"，而是让每个小团队都有清晰的任务、责任、权力和回报。
