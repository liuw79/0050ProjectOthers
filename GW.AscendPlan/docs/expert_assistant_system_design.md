# “启航图”专家辅助系统 (Expert Assistant System) 设计文档 V1.0

## 1. 系统定位与核心理念

根据用户反馈和补充材料，我们将“启航图”项目重新定位为一个**以“高维学堂”内部规划师为核心用户的专家辅助系统**。

- **核心理念**：**人机协同，专家驱动 (Human-AI Collaboration, Expert-Driven)**。
- **系统目标**：赋能规划师，而非取代规划师。通过技术手段将个性化的咨询服务过程标准化、数据化、智能化，从而提升服务效率、保证方案质量、沉淀组织智慧。
- **客户体验**：对于最终客户（企业创始人），他们感知到的是与专业规划师的深度对话和服务。系统的大部分功能对客户不可见，只在最终方案交付时体现其价值。

---

## 2. 核心业务流程

系统将支撑以下核心服务流程：

```mermaid
graph TD
    A[客户在线预约] --> B{规划师接受预约};
    B --> C[进行在线/线下访谈];
    C --> D[上传访谈记录(音/视频/文本)];
    D --> E{系统智能处理};
    E --> F[生成访谈洞察报告];
    F --> G{规划师结合洞察与知识库进行诊断};
    G --> H[撰写/生成学习规划方案];
    H --> I[向客户交付方案];
    I --> J[客户反馈与迭代];
```

## 3. 核心功能模块规划

### 模块一：客户预约与管理 (Client & Scheduling) - **【MVP新增模块】**

- **价值**：简化规划师的行政工作。
- **关键功能点**：
    - **客户信息管理**：记录企业基本信息、联系人、过往咨询记录等。
    - **预约日历**：规划师在后台设置可预约的时间段（区分线上/线下）。
- **客户预约界面**：生成一个对外的预约链接，客户打开后可以选择规划师、服务类型（线上/线下）、时间，并填写基本信息后提交预约。
- **预约通知**：预约成功后，自动向规划师和客户发送邮件或短信通知。
    - **状态追踪**：清晰展示每个客户处于“待接触”、“访谈中”、“方案交付”等哪个阶段。

### 模块二：访谈洞察引擎 (Interview Insight Engine) - **【MVP核心 & LLM驱动】**

- **价值**：利用大型语言模型(LLM)的强大能力，从非结构化的对话信息中，快速、精准地提炼核心问题与关键事实，将规划师从繁重的记录整理和信息挖掘中解放出来，直接提供初步洞察。
- **关键功能点 (V1)**:
    - **文本上传**：支持直接粘贴或上传 `.txt`、`.md` 格式的访谈文字稿。
    - **LLM驱动的关键信息提取**：通过调用公司现有的fastGPT服务（集成DeepSeek、千问等），利用精心设计的Prompt，实现对访谈稿的结构化信息提取。提取字段包括但不限于：个人背景、学业情况、兴趣特长、目标、困惑等。
    - **LLM驱动的初步洞察生成**：在结构化信息的基础上，再次调用LLM，生成一段简明扼要的洞察摘要，点出学生的核心优势、潜在风险和关键问题。
    - **洞察报告**：将提取的结构化信息和生成的洞察摘要，以清晰、可视化的方式呈现给规划师。
- **技术实现**: 详细技术方案请参考 **《访谈洞察引擎 - 技术设计文档》**。

### 模块三：诊断知识库 (Diagnostic Knowledge Base)

- **价值**：沉淀组织智慧，将顶尖规划师的经验转化为可复用的资产，帮助新规划师快速成长。
- **价值**：
    - **案例库**：存储过往的“客户背景-访谈洞察-诊断结论-解决方案”的完整案例。
    - **智能检索**：规划师输入当前客户遇到的问题关键词（如“高管离职”、“现金流紧张”），系统可智能推荐相似案例和解决方案框架。
    - **最佳实践沉淀**：定期组织规划师复盘，将成功的诊断思路和方案模板固化到知识库中。

### 模块四：方案生成器 (Proposal Generator)

- **价值**：提升方案撰写的效率和规范性。
- **关键功能点**：
    - **模板库**：内置多种针对不同行业、不同问题的学习规划方案模板。
    - **内容智能填充**：可一键引用“访谈洞察引擎”提取的关键信息和“知识库”中的相关建议，填充到方案模板中。
    - **版本管理与导出**：支持方案的多版本迭代，并可导出为 Word, PDF 等格式，方便交付。

## 4. 技术选型初步思考

- **前端**：考虑到是内部系统，初期可采用成熟的后台管理框架，如 `Vue + Element Plus` 或 `React + Ant Design`，快速开发。
- **后端**：`Python` 是自然的选择，因为它在数据处理和AI领域有强大的生态。可使用 `Django` 或 `Flask` 作为Web框架。
- **数据库**：`PostgreSQL`，功能强大，能很好地支持结构化数据和未来可能的JSONB数据存储。
- **核心引擎 (LLM)**：**V1版本直接采用公司现有的 `fastGPT` 服务**，通过API调用其集成的 `DeepSeek`、`千问` 等模型。不再采用传统的 `spaCy` 等NLP库作为核心。
- **日历与通知**：预约功能可能需要集成第三方日历服务（如Google Calendar API）或邮件/短信服务。

## 5. 产品路线图 (Roadmap)

根据我们对业务价值和技术可行性的判断，制定以下三步走的产品发展路线图：

- **V1 (MVP): 聚焦LLM驱动的核心价值**
    - **核心模块**: `客户预约与管理` + `访谈洞察引擎（LLM驱动版）`
    - **核心目标**: 验证LLM在深刻理解访谈内容、提升分析效率与深度上的核心价值，为规划师提供真正有吸引力的智能辅助。
    - **状态**: **当前正在执行**

- **V2: 沉淀核心资产，构建智慧大脑**
    - **核心模块**: `诊断知识库` + `访谈洞察引擎（增强版）`
    - **核心目标**: 将专家的诊断经验和解决方案结构化、数字化，形成可复用的组织资产，并提升AI分析的深度和准度。

- **V3: 实现智能生成，放大专家产能**
    - **核心模块**: `方案生成器` + `持续优化的知识库与引擎`
    - **核心目标**: 基于知识库和洞察结果，智能生成个性化的方案初稿，将规划师从重复性的撰写工作中解放出来，聚焦于更高价值的创造性诊断和沟通。

## 6. 综合方案对比分析 (融合三方智慧)

为了确保当前方案的务实性和聚焦性，我们将其与 `Gemini`、`DeepSeek` 的方案以及用户的核心要求（`我的补充.txt`）进行综合对比分析。

| 对比维度 | Gemini & DeepSeek 方案 (共性) | 当前专家辅助方案 (V1.0) | 差异与取舍 |
| :--- | :--- | :--- | :--- |
| **核心定位** | 面向最终客户的、功能全面的SaaS学习平台 | 赋能内部规划师的专家辅助工具 | **根本性转变**：从“做平台”转向“做工具”，完美契合了“隐藏系统，凸显专家”的核心要求。 |
| **用户界面** | 复杂，功能繁多，需要客户深度参与和学习 | 极简，客户界面只有预约功能，核心功能由内部专家使用 | **体验聚焦**：将复杂性保留在内部，给予客户最简洁、最符合“人对人”咨询习惯的体验。 |
| **实现路径** | 倾向于一次性构建一个庞大的系统 | MVP优先，从“访谈洞察引擎”单点突破，快速迭代 | **风险控制**：从瀑布式开发转向敏捷开发，风险更低，能更快验证核心价值。 |
| **数据依赖** | 依赖课程库、岗位模型等结构化数据 | **核心依赖访谈记录（非结构化数据）** | **价值挖掘**：直接从最源头的、最有价值的客户访谈数据入手，构建核心壁垒。 |
| **核心技术** | 传统Web开发、LMS功能、推荐算法 | **NLP、ASR、知识图谱等AI技术** | **技术壁垒**：技术重心从“功能实现”转向“智能分析”，护城河更深。 |

### **当前方案的优势 (Pros):**

1.  **高度务实 (Pragmatic)**: 完美契合了“第一版系统不出现在客户界面上”的核心要求，聚焦于提升内部效率这一明确痛点。
2.  **风险可控 (Risk-Controllable)**: 采用MVP（最小可行产品）思路，优先开发“访谈洞察引擎”这一核心模块，可以快速验证技术可行性和业务价值，避免了早期大规模投入带来的沉没成本风险。
3.  **壁垒更高 (Higher Barrier to Entry)**: 系统的核心竞争力在于对“访谈”这一非结构化数据的深度理解和处理能力，以及在此基础上沉淀的“诊断知识库”。这比做一个功能性的LMS平台更难被模仿，是真正的护城河。
4.  **数据驱动 (Data-Driven at Core)**: 从第一天起，系统就在处理和结构化最有价值的数据——创始人的真实问题和专家的解决方案。这为未来的智能化升级和产品演进积累了最宝贵的资产。

| 对比维度 | 早期宏大方案 (假设) | 当前专家辅助方案 (V1.0) |
| :--- | :--- | :--- |
| **核心定位** | 面向最终客户的SaaS学习平台 | 赋能内部规划师的专家辅助工具 |
| **用户界面** | 复杂，功能繁多，需要客户学习使用 | 极简，客户界面只有预约功能 |
| **实现难度** | 极高，需要庞大的前后端开发团队 | 相对可控，可分模块、MVP方式迭代 |
| **开发周期** | 漫长，风险高 | 短，可快速验证核心价值 |
| **数据依赖** | 需要大量结构化数据和课程内容 | **核心依赖访谈记录（非结构化数据）** |
| **核心技术** | 传统Web开发、LMS功能 | **NLP、ASR、知识图谱等AI技术** |
| **价值闭环** | 线上完成学习、评估、反馈闭环 | **支撑“人”的服务流程，线下完成价值闭环** |

### **当前方案的优势 (Pros):**

1.  **高度务实 (Pragmatic)**: 完美契合了“第一版系统不出现在客户界面上”的核心要求，聚焦于提升内部效率这一明确痛点。
2.  **风险可控 (Risk-Controllable)**: 采用MVP（最小可行产品）思路，优先开发“访谈洞察引擎”这一核心模块，可以快速验证技术可行性和业务价值，避免了早期大规模投入带来的沉没成本风险。
3.  **壁垒更高 (Higher Barrier to Entry)**: 系统的核心竞争力在于对“访谈”这一非结构化数据的深度理解和处理能力，以及在此基础上沉淀的“诊断知识库”。这比做一个功能性的LMS平台更难被模仿，是真正的护城河。
4.  **数据驱动 (Data-Driven at Core)**: 从第一天起，系统就在处理和结构化最有价值的数据——创始人的真实问题和专家的解决方案。这为未来的智能化升级和产品演进积累了最宝贵的资产。

### **当前方案的潜在挑战 (Cons & Challenges):**

1.  **对NLP技术依赖高 (High Dependency on NLP)**: “访谈洞察引擎”的效果直接决定了系统的成败。如果关键信息提取的准确率不高，会严重影响规划师的使用体验和效率。
2.  **冷启动问题 (Cold Start Problem)**: “诊断知识库”在初期是空的，需要有意识地进行案例积累和知识沉淀，才能发挥其威力。需要建立配套的知识管理流程。
3.  **对规划师的要求更高 (Higher Demands on Planners)**: 系统是辅助工具，不是全自动方案生成器。它要求规划师能理解并有效利用AI的洞察结果，结合自身专业知识进行最终诊断。这需要对规划师进行相应的培训。

### **结论：融合后的专家辅助方案是当前阶段的最佳选择**

融合了各方智慧的“专家辅助方案 V1.0”是当前阶段最明智的选择。它深刻理解并采纳了用户“隐藏系统，凸显专家”的核心思想，将Gemini和DeepSeek方案中宏大的“平台梦”作为长期愿景，但将第一步牢牢地踩在了“赋能专家、提升单点服务效率与质量”这个最坚实的地面上。

它虽然在功能广度上有所收缩，但在**深度、可行性和长期价值**上远超任何单一方案。它是一个更聪明、更聚焦、风险更低的起点，完全符合“保持务实”的原则。我们应该坚定地沿着这个方向推进。
