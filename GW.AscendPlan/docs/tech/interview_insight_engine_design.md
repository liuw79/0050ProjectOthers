# 访谈洞察引擎 - 技术设计文档

## 1. 概述

本文档旨在详细阐述“启航图”专家辅助系统中核心模块——**访谈洞察引擎 (Interview Insight Engine)** 的技术实现方案。根据最新项目决策，V1 (MVP) 版本将直接集成大型语言模型 (LLM) 以提供强大的自然语言处理能力，从而确保产品对规划师的核心吸引力。

## 2. V1 (MVP) 核心目标与技术选型

### 2.1. 核心目标

*   **自动化关键信息提取**：从客户访谈的文字稿中，自动识别并结构化提取关键信息，如个人背景、学业情况、兴趣特长、未来期望、主要困惑等。
*   **初步洞察生成**：基于提取的信息，利用LLM进行初步的分析，生成简要的洞察摘要，帮助规划师快速把握客户的核心问题和需求。
*   **降本增效**：将规划师从繁琐的手动信息整理和初步分析中解放出来，显著提升工作效率和标准化水平。

### 2.2. 技术架构

V1将采用基于 **RAG (Retrieval-Augmented Generation)** 的简化架构，充分利用公司现有的 **fastGPT** 服务平台。

```mermaid
graph TD
    A[用户上传访谈文字稿] --> B{访谈洞察引擎};
    B --> C[1. 文本预处理];
    C --> D[2. 关键信息提取 (LLM Call)];
    D --> E{3. 初步洞察生成 (LLM Call)};
    F[诊断知识库 (V2规划)] -.-> D;
    F -.-> E;
    E --> G[4. 结构化数据输出];
    G --> H[前端展示];

    subgraph fastGPT 服务
        I(DeepSeek / 千问等模型)
    end

    D -- Prompt 1 --> I;
    E -- Prompt 2 --> I;
    I -- JSON/Text --> D;
    I -- Text --> E;
```

### 2.3. 技术栈

*   **核心引擎**: Python
*   **LLM服务**: 公司内部署的 **fastGPT** 平台 (通过API调用)
*   **核心模型**: DeepSeek, 千问 (Qwen) 或其他fastGPT集成的模型，根据效果择优。
*   **数据交换格式**: 后端与LLM服务、后端与前端之间主要使用 JSON 进行数据交换。

## 3. 实现流程详解

### 3.1. 文本预处理

*   **输入**: 用户上传的访谈文稿 (纯文本格式)。
*   **处理**: 进行基础的文本清洗，如去除不必要的空格、特殊字符，规范化段落等，为LLM处理做好准备。

### 3.2. 关键信息提取 (Prompt Engineering)

这是V1的核心步骤。我们将设计一套高效的 **Prompt**，引导LLM完成结构化信息提取任务。

*   **方法**: 采用 few-shot learning 的方式，在Prompt中提供1-2个高质量的提取示例，引导LLM输出稳定、格式化的JSON数据。
*   **Prompt示例 (概念)**:

    ```text
    你是一个专业的升学规划访谈分析师。请从以下访谈记录中，提取关键信息，并以JSON格式输出。请严格按照指定的字段进行提取，如果信息不存在，请将字段值留空。

    要提取的字段：
    - "name": "学生姓名"
    - "grade": "年级"
    - "school": "就读学校"
    - "strengths": ["学科优势", "其他特长"]
    - "weaknesses": ["学科弱点"]
- "interests": ["兴趣爱好"]
    - "goals": ["短期目标", "长远理想"]
    - "concerns": ["主要困惑和担忧"]

    --- 示例 ---
    访谈记录: "..."
    输出: {"name": "小明", ...}
    --- 结束示例 ---

    --- 正式任务 ---
    访谈记录: "{待处理的访谈文本}"
    输出:
    ```

*   **输出**: 一个包含所有关键信息的结构化JSON对象。

### 3.3. 初步洞察生成

在提取出结构化信息后，进行第二次LLM调用，以生成更具可读性的洞察摘要。

*   **方法**: 将上一步提取的JSON数据，连同一个新的Prompt，发送给LLM。
*   **Prompt示例 (概念)**:

    ```text
    你是一名资深升学规划专家。请根据以下从学生访谈中提取的关键信息，撰写一段150字左右的初步洞察分析。分析需要点出学生的核心优势、潜在风险以及最迫切需要解决的问题。

    关键信息如下：
    {第一步生成的JSON数据}

    你的分析：
    ```

*   **输出**: 一段自然语言文本，作为洞察摘要，展示给规划师。

## 4. 后续版本规划 (V2/V3)

*   **V2 - 引入RAG与知识库**: 正式构建“诊断知识库”，将知识库内容作为LLM的检索源，使洞察分析不再局限于单次访谈内容，而是结合更广泛的专业知识，提升分析的深度和准确性。
*   **V3 - 微调 (Fine-tuning) LLM**: 在积累了足够多的高质量“访谈记录-洞察报告”数据对后，对特定模型进行微调，以打造专属于“启航图”的、更懂升学规划的垂直领域模型，最终实现高质量“学习规划方案”的智能生成。
