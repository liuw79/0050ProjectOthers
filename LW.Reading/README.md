# 考试系统 - 管理的实践

基于德鲁克《管理的实践》第23章内容开发的考试系统，支持多种题型和自动评分功能。

## 功能特点

### 📚 题目管理
- **多种题型支持**：单选题、多选题、填空题
- **分层目录结构**：按章节和主题组织题目
- **难度分级**：1-5级难度设置
- **详细解析**：每道题目都有详细的解答说明

### 🎯 考试功能
- **学生信息管理**：记录考生姓名和考试信息
- **分类考试**：可选择不同主题进行专项考试
- **实时保存**：答题过程中自动保存答案
- **灵活导航**：支持前进后退，随时修改答案

### 📊 评分系统
- **自动评分**：系统自动计算得分
- **详细反馈**：显示每道题的对错情况
- **答案解析**：提供正确答案和详细解释
- **成绩记录**：保存考试历史记录

### 💾 数据存储
- **SQLite数据库**：轻量级本地数据库存储
- **完整记录**：保存题目、答案、成绩等所有信息
- **数据持久化**：考试记录永久保存

## 系统架构

```
考试系统/
├── ExamDatabase      # 数据库管理类
│   ├── 分类管理
│   ├── 题目管理
│   └── 成绩记录
├── ExamSystem        # 考试逻辑类
│   ├── 题目初始化
│   ├── 考试流程控制
│   └── 评分计算

```

## 数据库结构

### 分类表 (categories)
- `id`: 分类ID（主键）
- `name`: 分类名称
- `parent_id`: 父分类ID（支持层级结构）
- `description`: 分类描述
- `created_at`: 创建时间

### 题目表 (questions)
- `id`: 题目ID（主键）
- `category_id`: 所属分类
- `question_type`: 题目类型（single/multiple/fill）
- `question_text`: 题目内容
- `options`: 选项（JSON格式）
- `correct_answer`: 正确答案（JSON格式）
- `explanation`: 答案解析
- `difficulty`: 难度等级（1-5）
- `created_at`: 创建时间

### 考试记录表 (exam_records)
- `id`: 记录ID（主键）
- `student_name`: 学生姓名
- `category_id`: 考试分类
- `total_questions`: 总题数
- `correct_answers`: 正确题数
- `score`: 得分
- `start_time`: 开始时间
- `end_time`: 结束时间
- `answers`: 答题详情（JSON格式）

## 使用方法

### 1. 启动系统
```bash
python exam_system.py
```

### 2. 开始考试
1. 输入学生姓名
2. 选择考试分类
3. 点击"开始考试"

### 3. 答题过程
- **单选题**：点击选择一个答案
- **多选题**：勾选多个正确答案
- **填空题**：在文本框中输入答案
- 使用"上一题"、"下一题"按钮导航
- 最后一题完成后点击"提交考试"

### 4. 查看结果
- 系统自动计算得分
- 显示每道题的对错情况
- 提供详细的答案解析
- 可选择重新开始考试

## 开发说明

本系统采用模块化设计，代码结构清晰，便于维护和扩展。主要模块包括：

- **数据层**：ExamDatabase类负责所有数据库操作
- **业务层**：ExamSystem类处理考试逻辑和评分算法
- **表现层**：ExamGUI类提供用户交互界面

每个模块都有明确的职责分工，遵循单一职责原则，确保代码的可维护性和可扩展性。
