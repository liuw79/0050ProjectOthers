# 写作参考 - 使用说明

这个文件夹存放你的个人信息、风格指南和代表性文章，用于AI学习你的写作风格。

## 需要创建的文件

### 1. 个人信息.md（必需）

创建这个文件，包含以下内容：

```markdown
# 个人信息

## 基本信息
- 姓名/笔名：[你的名字]
- 职业：[你的职业]
- 专长领域：[你擅长的领域，如AI、产品开发、创业等]
- 写作年限：[多久了]

## 写作定位
- 主要写作主题：[你主要写什么]
- 目标受众：[谁会读你的文章]
- 写作平台：[公众号、知乎、小红书等]

## 个人特点
- 职业背景：[简要描述]
- 技术栈/专业技能：[你会什么]
- 代表项目/成就：[做过什么值得一提的]

## 写作目标
- 为什么写作：[动机]
- 希望传达什么：[核心价值观]
```

### 2. 风格指南.md（必需）

创建这个文件，包含以下内容：

```markdown
# 写作风格指南

## 语言特征

### 口语化程度
- [ ] 非常口语化（像聊天）
- [ ] 较口语化（轻松自然）
- [ ] 适中（平衡书面和口语）
- [ ] 偏书面（正式严谨）

### 句式偏好
- 平均句长：[15-20字 / 20-30字 / 其他]
- 常用句式：[短句为主 / 长短结合 / 其他]
- 段落长度：[手机屏幕3行 / 5行 / 其他]

### 常用词汇和表达
列举你常用的词汇、口头禅：
- 示例："确实"、"说实话"、"但问题是"

## 文章结构

### 开头方式（按偏好排序）
1. [ ] 真实经历/故事
2. [ ] 直接抛出观点
3. [ ] 场景描述
4. [ ] 数据/现象
5. [ ] 提问

### 结构偏好
- [ ] 总分总（先观点，再论证，最后总结）
- [ ] 并列式（多个角度平行论述）
- [ ] 递进式（层层深入）
- [ ] 故事式（用故事串联）

### 结尾方式
- [ ] 总结升华
- [ ] 行动建议
- [ ] 留下思考
- [ ] 个人感悟

## 风格特点

### 金句使用
- [ ] 经常用金句（每篇2-3句）
- [ ] 偶尔用金句
- [ ] 不刻意追求金句

### 案例使用
- [ ] 大量真实案例（占比50%+）
- [ ] 适量案例（占比30%）
- [ ] 少量案例（占比10%）

### 数据使用
- [ ] 数据驱动（大量数据支撑）
- [ ] 适量数据（关键处使用）
- [ ] 少用数据（以观点和故事为主）

## 禁用词汇和套话清单

### 绝对不用的词汇
- "在当今时代"
- "综上所述"
- "值得注意的是"
- [补充你不喜欢的表达]

### 避免的句式
- "不是...而是..."连续出现
- "一方面...另一方面..."
- [补充其他]

## 其他偏好

### 标点符号
- [ ] 多用句号（一句话就是一句话）
- [ ] 适量逗号（连接相关内容）
- [ ] 常用问号（引发思考）
- [ ] 使用破折号（补充说明）

### 排版习惯
- [ ] 多用小标题（方便阅读）
- [ ] 适量加粗（突出重点）
- [ ] 使用列表（清晰呈现）
- [ ] 留白充足（不拥挤）
```

### 3. 信息源.md（可选但推荐）

列出你常用的信息来源：

```markdown
# 常用信息源

## 科技媒体
- TechCrunch
- The Verge
- [你常看的其他媒体]

## 社区论坛
- Hacker News
- Reddit (r/programming, r/MachineLearning)
- [其他]

## 个人博客/Newsletter
- [你关注的人]

## 其他来源
- [补充]
```

### 4. 代表作文件夹（必需）

创建 `代表作/` 文件夹，放入2-3篇你最满意的文章（Markdown格式）。

**文件命名：**
- `文章标题.md`

**内容要求：**
- 纯文本（Markdown格式）
- 保留原文的格式、标点、用词
- 这些文章应该代表你的真实水平和风格

---

## 使用方法

AI会在写作前自动：
1. 读取 `个人信息.md` 了解你是谁
2. 读取 `风格指南.md` 学习你的风格偏好
3. 读取 `代表作/` 中的文章，提取你的语言特征
4. 在创作时模仿你的风格

---

## 更新建议

- 每写完一篇满意的文章，考虑加入 `代表作/`
- 发现新的风格特点，及时更新 `风格指南.md`
- 定期回顾和优化（建议每季度一次）
