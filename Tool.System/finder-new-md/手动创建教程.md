# 手动创建 Finder 右键新建 MD 文件 - 图文教程

## 🎯 这是最可靠的方法！

不依赖任何自动脚本，直接在 Automator 中创建。

---

## 📝 详细步骤

### 步骤 1：打开 Automator

1. 按 `⌘ + 空格` 打开聚焦搜索
2. 输入 `Automator`
3. 回车打开

### 步骤 2：创建新的快速操作

1. 在弹出的窗口中选择 **「快速操作」**
2. 点击「选取」按钮

### 步骤 3：配置工作流设置

在窗口顶部，设置：
- **「工作流程接收当前」** 选择：**没有输入**
- **「位于」** 选择：**Finder.app**

### 步骤 4：添加 Shell 脚本操作

1. 在左侧搜索框输入：`shell`
2. 找到 **「运行 Shell 脚本」** 操作
3. **双击** 或 **拖动** 到右侧工作区

### 步骤 5：配置脚本

在右侧的脚本编辑区：

1. **Shell** 下拉菜单选择：`/bin/bash`

2. **传递输入** 选择：`作为自变量`

3. **删除默认代码**，粘贴以下内容：

```bash
#!/bin/bash

# 获取当前 Finder 目录
if [ -n "$1" ]; then
    target_dir="$1"
else
    target_dir=$(osascript -e 'tell application "Finder" to POSIX path of (insertion location as alias)')
fi

# 如果是文件，获取其父目录
if [ -f "$target_dir" ]; then
    target_dir=$(dirname "$target_dir")
fi

# 确保路径存在
if [ ! -d "$target_dir" ]; then
    osascript -e 'display notification "无法确定目标目录" with title "新建 Markdown 文件"'
    exit 1
fi

# 生成唯一文件名
base_name="未命名"
extension=".md"
counter=1
file_name="${base_name}${extension}"
file_path="${target_dir}/${file_name}"

while [ -e "$file_path" ]; do
    file_name="${base_name}${counter}${extension}"
    file_path="${target_dir}/${file_name}"
    ((counter++))
done

# 创建文件
cat > "$file_path" << 'EOF'
# 标题

## 简介

内容...

## 详细信息

EOF

# 在 Finder 中显示并选中
osascript <<APPLESCRIPT
    tell application "Finder"
        activate
        reveal POSIX file "$file_path"
        select POSIX file "$file_path"
    end tell
APPLESCRIPT

# 显示通知
osascript -e "display notification \"已创建: ${file_name}\" with title \"新建 Markdown 文件\""

# 进入重命名模式
sleep 0.3
osascript -e 'tell application "System Events" to keystroke return'
```

### 步骤 6：保存

1. 按 `⌘S` 或点击菜单「文件」→「存储」
2. 名称输入：`新建 Markdown 文件`
3. 位置会自动设置为「服务」文件夹
4. 点击「存储」

### 步骤 7：测试

1. 关闭 Automator
2. 打开 Finder，进入任意文件夹
3. 在空白处右键点击
4. 选择 **「快速操作」→「新建 Markdown 文件」**

---

## 🎬 预期效果

- 会创建一个新的 `.md` 文件
- 文件名默认为「未命名.md」
- 自动进入重命名状态
- 显示系统通知

---

## ⚠️ 注意事项

### 如果看不到「快速操作」菜单：

1. **等待几秒**：系统需要时间识别
2. **重启 Finder**：`Option + 右键点击 Finder 图标` → 「重新启动」
3. **注销重新登录**：最可靠的方法

### 如果脚本粘贴后格式错乱：

1. 先粘贴到文本编辑器
2. 确保是纯文本格式
3. 再复制到 Automator

### 如果执行报错：

检查脚本开头是否有：
```bash
#!/bin/bash
```
