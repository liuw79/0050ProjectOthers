# Finder 快速操作无法显示 - 故障排除指南

## 问题：重启 Finder 后仍然看不到「新建 Markdown 文件」

这是 macOS 的常见问题，系统需要时间来识别新的服务。以下是按效果排序的解决方案：

---

## 🎯 方案 1：在 Automator 中保存一次（推荐）✨

**最有效且不需要重启**

```bash
cd /Users/comdir/SynologyDrive/0050Project/Tool.System/finder-new-md
./refresh_service.sh
```

然后：
1. 等待 Automator 自动打开
2. 直接按 `⌘S` 保存
3. 关闭 Automator
4. 回到 Finder 右键测试

**原理**：通过 Automator 保存，系统会立即注册服务

---

## 🎯 方案 2：注销并重新登录（最可靠）

**100% 有效，但需要重新登录**

1. 点击  菜单
2. 选择「注销...」
3. 重新登录
4. 打开 Finder 测试

**原理**：注销时系统会重新加载所有服务

---

## 🎯 方案 3：使用 AppleScript 版本（替代方案）

**如果上述方法都不行，使用纯 AppleScript 实现**

```bash
cd /Users/comdir/SynologyDrive/0050Project/Tool.System/finder-new-md
./install_applescript_version.sh
```

这会创建一个名为「新建 Markdown 文件 v2」的快速操作。

然后在 Automator 中保存一次（同方案 1）。

---

## 🎯 方案 4：手动通过 Automator 创建（万无一失）

如果所有自动化方式都失败，手动创建最可靠：

### 步骤：

1. **打开 Automator**
   - 应用程序 → Automator

2. **创建新文稿**
   - 选择「快速操作」

3. **配置工作流设置**
   - 「工作流程接收当前」→ 选择「没有输入」
   - 「位于」→ 选择「Finder.app」

4. **添加操作**
   - 左侧搜索「运行 Shell 脚本」
   - 拖到右侧工作区

5. **输入脚本内容**
   ```bash
   "/Users/comdir/SynologyDrive/0050Project/Tool.System/finder-new-md/create_new_md.sh" "$@"
   ```

6. **配置脚本选项**
   - Shell: `/bin/bash`
   - 传递输入: `作为自变量`

7. **保存**
   - 按 `⌘S`
   - 名称: `新建 Markdown 文件`
   - 位置会自动设置为 `~/Library/Services`

8. **立即生效**
   - 关闭 Automator
   - 在 Finder 右键测试

---

## 🔍 验证安装

运行以下命令检查：

```bash
# 检查文件是否存在
ls -la ~/Library/Services/ | grep -i markdown

# 应该看到类似输出：
# drwxr-xr-x  3 user  staff  96 Feb  1 11:43 新建 Markdown 文件.workflow
```

## 🐛 其他可能的问题

### 问题 1：权限问题

```bash
# 给予完全权限
sudo chmod -R 755 ~/Library/Services/新建\ Markdown\ 文件.workflow
```

### 问题 2：脚本路径问题

检查 `create_new_md.sh` 是否有执行权限：

```bash
ls -la /Users/comdir/SynologyDrive/0050Project/Tool.System/finder-new-md/create_new_md.sh

# 应该看到 -rwxr-xr-x（包含 x）
```

如果没有：

```bash
chmod +x /Users/comdir/SynologyDrive/0050Project/Tool.System/finder-new-md/create_new_md.sh
```

### 问题 3：macOS 版本兼容性

某些旧版本 macOS 可能需要：

1. 打开「系统偏好设置」→「安全性与隐私」
2. 点击「隐私」标签
3. 左侧选择「自动化」
4. 确保 Automator 有权限
