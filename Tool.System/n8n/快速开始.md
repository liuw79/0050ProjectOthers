# N8N 快速开始指南

## 🚀 两步启动 N8N

### 第一步：确保 Docker 已安装并运行

1. 如果还没有安装 Docker，请下载安装：
   - 下载地址: https://www.docker.com/products/docker-desktop
   - 安装后启动 Docker Desktop

2. 确认 Docker 正在运行：
   - 在菜单栏看到 Docker 图标即可

### 第二步：启动 N8N

**方式一：双击启动脚本（推荐）**
```
双击 "启动N8N.command"
```

**方式二：命令行启动**
```bash
cd /Users/comdir/SynologyDrive/0050Project/Tool.System/n8n
docker-compose up -d
```

等待几秒钟后，浏览器会自动打开 http://localhost:5678

## 📝 基本操作

### 启动服务
```bash
双击 "启动N8N.command"
```

### 停止服务
```bash
双击 "停止N8N.command"
```

### 查看日志
```bash
双击 "查看日志.command"
```

### 修改密码

1. 停止服务
2. 编辑 `.env` 文件
3. 修改这两行：
   ```env
   N8N_BASIC_AUTH_USER=你的用户名
   N8N_BASIC_AUTH_PASSWORD=你的密码
   ```
4. 重新启动服务

## 🎯 第一个工作流

1. 访问 http://localhost:5678 并登录
2. 点击 "Create new workflow"
3. 添加一个 "Schedule" 节点（定时触发）
4. 添加一个 "HTTP Request" 节点（发送请求）
5. 连接两个节点
6. 点击 "Active" 激活工作流

## 💡 常用功能

### 1. Webhook 接收
创建一个 Webhook 节点，获取自己的 Webhook URL，可以接收外部请求

### 2. 定时任务
使用 Schedule 或 Cron 节点，设置定时执行的任务

### 3. 数据处理
使用 Function 节点编写 JavaScript 代码处理数据

### 4. API 集成
使用 HTTP Request 节点调用任何 API

### 5. 数据库操作
可以连接 MySQL、PostgreSQL、MongoDB 等数据库

## ❓ 遇到问题？

### Docker 未运行
启动 Docker Desktop 应用

### 端口 5678 被占用
修改 `docker-compose.yml` 中的端口映射：
```yaml
ports:
  - "5679:5678"  # 改为其他端口
```

### 忘记密码
1. 停止服务: `docker-compose down`
2. 修改 `.env` 文件中的密码
3. 重启服务: `docker-compose up -d`

### 数据丢失
数据保存在 `n8n_data/` 目录，定期备份这个目录即可

## 🔄 更新 N8N

```bash
# 拉取最新版本
docker-compose pull

# 重启服务
docker-compose up -d
```

## 📦 备份与恢复

### 备份
```bash
# 备份数据目录
tar -czf n8n_backup_$(date +%Y%m%d).tar.gz n8n_data/
```

### 恢复
```bash
# 解压备份
tar -xzf n8n_backup_20250101.tar.gz
```
