# 飞书邮件发送技能 - 学习总结

## 已完成尝试

### 1. 浏览器控制方案
- ✅ 成功打开飞书邮箱
- ✅ 成功填写邮件内容（主题、正文）
- ❌ 收件人格式问题（无法识别"刘军"，需从联系人列表选择）
- ❌ 浏览器连接不稳定（易断开）

### 2. 发现的关键问题

**收件人格式**：
- 飞书邮箱不直接接受文本输入的邮箱地址
- 需要从联系人列表中选择（有自动补全功能）
- 格式必须是系统中已存在的联系人

**稳定性问题**：
- Chrome扩展attach方式容易断开
- 每次操作后需要重新确认连接状态

## 更优方案

### 方案1：飞书邮件API（推荐）
```javascript
// 使用飞书开放平台邮件API
POST /open-apis/mail/v1/messages
{
  "subject": "马年新春祝福",
  "body": "...",
  "to": ["user1@gaowei.com", "user2@gaowei.com"]
}
```
**优势**：无需浏览器，稳定可靠
**需申请**：mail:message:send 权限

1. **不要过早宣布成功** - 必须验证"已发送"才确认

4. **API > 自动化** - 有API优先用API，而不是模拟点击
