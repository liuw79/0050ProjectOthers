# 🔧 电池控制密码问题解决方案

## 问题描述

当你按 `Cmd+Alt+B` 快捷键时，出现了 sudo 密码错误：
```
sudo: a terminal is required to read the password; either use the -S option to read from standard input or configure an askpass helper
```

## 问题原因

- `batt` 命令需要管理员权限来控制电池充电限制
- 在 Hammerspoon 环境中，无法交互式输入 sudo 密码
- 原脚本使用 `sudo` 命令，但没有处理密码输入问题

## 解决方案

我已经修改了 `battery_control_wrapper.sh` 脚本，使用 `osascript` 来处理密码输入：

### 修改内容

1. **添加了 `run_with_sudo` 函数**：
   ```bash
   run_with_sudo() {
       local cmd="$1"
       osascript -e "do shell script \"$cmd\" with administrator privileges"
   }
   ```

2. **替换所有 sudo 调用**：
   - `sudo "$BATT_PATH" status` → `run_with_sudo "$BATT_PATH status"`
   - `sudo "$BATT_PATH" limit 80` → `run_with_sudo "$BATT_PATH limit 80"`
   - `sudo "$BATT_PATH" limit 100` → `run_with_sudo "$BATT_PATH limit 100"`

### 工作原理

- `osascript` 会弹出系统密码对话框
- 用户输入密码后，命令会以管理员权限执行
- 这样就避免了终端密码输入的问题

## 测试结果

✅ 脚本现在可以正常工作：
```bash
$ ./battery_control_wrapper.sh toggle
🔋 电池充电限制已设为80%
```

## 使用方法

现在你可以正常使用快捷键了：

1. **主要快捷键：`Ctrl+Alt+B`**
2. **备用快捷键：`Cmd+Alt+B`**
3. **第三选择：`Ctrl+Shift+F12`**

当你按下快捷键时：
1. 系统会弹出密码对话框
2. 输入你的管理员密码
3. 电池充电限制会在 80% 和 100% 之间切换
4. 会显示操作结果的提示信息

## 注意事项

- 首次使用时需要输入管理员密码
- 密码会被系统记住一段时间（通常几分钟）
- 如果不想每次都输入密码，可以考虑配置 sudo 免密（不推荐，有安全风险）

现在请再次尝试按 `Ctrl+Alt+B` 或 `Cmd+Alt+B` 来测试电池控制功能！
