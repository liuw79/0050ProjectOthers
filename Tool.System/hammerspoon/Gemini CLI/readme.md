# Gemini CLI 使用指南

Gemini CLI 是一个命令行工具，可以让你与 Google Gemini 模型进行交互。

## 安装

运行 `install_gemini_cli.ps1` 脚本即可完成安装。该脚本会自动检查 Node.js 环境并从 npm 安装 Gemini CLI。

## 配置与使用

安装成功后，你有两种方式进行配置和使用：

### 1. 浏览器登录 (推荐)

这是最简单的方式。

1.  打开一个新的 PowerShell 或终端窗口。
2.  输入 `gemini` 命令。
3.  一个浏览器窗口会自动弹出，引导你使用 Google 账户登录授权。

### 2. 使用 API 密钥

适用于需要进行自动化或高级脚本编程的场景。

1.  **获取API密钥**: 访问 [Google AI Studio](https://aistudio.google.com/app/apikey) 创建并获取你的 API 密钥。

2.  **设置环境变量**: 

    **[重要安全警告]**：**绝对不要**将你的 API 密钥直接写入任何脚本文件或纯文本文件中。这会带来严重的安全风险。请始终使用环境变量来存储密钥。

    在 PowerShell 中运行以下命令来设置临时环境变量（只在当前会话有效）：

    ```powershell
    $env:GEMINI_API_KEY="此处替换为你的API密钥"
    ```

    要使环境变量永久生效，请将其添加到你的系统环境变量或 PowerShell 配置文件 (`$PROFILE`) 中。

## 疑难解答：`gemini` 命令无法识别

如果在安装后，在新的 PowerShell 终端中输入 `gemini` 命令，系统提示“无法将‘gemini’项识别为 cmdlet、函数、脚本文件或可运行程序的名称”，这通常意味着 `npm` 的全局安装路径没有被正确添加到系统的 `PATH` 环境变量中。您可以按照以下步骤手动修复：

1.  **打开环境变量设置**：
    *   按下 `Win` + `R` 键，打开“运行”对话框。
    *   输入 `sysdm.cpl` 并按回车，打开“系统属性”。
    *   切换到“高级”选项卡，然后点击“环境变量”按钮。

2.  **编辑用户变量 `Path`**：
    *   在“用户变量”区域，找到并选中名为 `Path` 的变量，然后点击“编辑”。

3.  **添加 `npm` 路径**：
    *   在“编辑环境变量”窗口中，点击“新建”。
    *   在输入框中，粘贴以下路径：`%APPDATA%\npm`
    *   点击“确定”保存更改。

4.  **保存并重启**：
    *   一路点击“确定”关闭所有设置窗口。
    *   **非常重要**：关闭所有已打开的 PowerShell 或命令提示符窗口，然后重新打开一个新的 PowerShell 窗口。只有这样，新的环境变量才会生效。

5.  **验证**：
    *   在新的 PowerShell 窗口中，再次尝试运行 `gemini` 命令，此时应该可以正常工作了。

---

## 基本用法

配置完成后，你可以通过 `gemini` 命令与模型进行交互。例如，向模型提问：

```bash
gemini "中国的首都是哪里？"
```

更多高级用法，请参考官方文档或使用 `gemini --help` 查看所有可用命令。
