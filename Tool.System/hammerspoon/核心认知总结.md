# Hammerspoon 调试核心认知总结

## 状态管理的坑
**问题**：局部变量在配置重载时会重置，导致状态丢失
**解决**：使用 `_G.moduleName` 全局对象持久化状态
```lua
-- 错误做法
local isTemporaryCharge = false

-- 正确做法
if not _G.batteryMonitorState then
    _G.batteryMonitorState = { isTemporaryCharge = false }
end
```

## 窗口关闭的兼容性
**问题**：不同应用的窗口关闭机制不同
**解决**：分应用类型处理 + 多重降级策略
- 系统应用：直接用 `Cmd+W`
- 普通应用：先试 `close()` 再试快捷键
- 最后兜底：菜单操作

## 配置重载的正确姿势
**问题**：`hs.reload()` 命令行调用失败
**解决**：用 `pkill + open` 强制重启
```bash
pkill -f Hammerspoon && sleep 2 && open -a Hammerspoon
```

## 调试思路
1. **先验证语法**：确保代码能加载
2. **再测试逻辑**：写独立测试脚本
3. **最后验证状态**：检查变量是否正确更新
4. **分层排查**：从模块到函数到变量逐层定位

## 关键教训
- **状态要持久化**：重载不能丢状态
- **兼容性要考虑**：不同应用行为不同  
- **调试要系统**：别急着改代码，先搞清楚问题
- **测试要独立**：写小脚本验证单个功能
