# Rustdesk 客户端配置指南

## 服务器信息
- **服务器地址**: op.gaowei.com
- **信令服务器端口**: 21115
- **中继服务器端口**: 21117

## 配置步骤

### 1. 下载 Rustdesk 客户端
- 官网: https://rustdesk.com/
- GitHub: https://github.com/rustdesk/rustdesk/releases

### 2. 配置网络设置

#### Windows/Mac/Linux 客户端:
1. 打开 Rustdesk 客户端
2. 点击右上角的 "⚙️" (设置) 按钮
3. 选择 "网络" 选项卡
4. 在 "ID服务器" 中输入: `op.gaowei.com:21115`
5. 在 "中继服务器" 中输入: `op.gaowei.com:21117`
6. 在 "公钥" 中输入服务器生成的公钥内容
7. 点击 "应用" 保存设置

#### Android/iOS 客户端:
1. 打开 Rustdesk 应用
2. 点击 "设置" -> "网络"
3. 设置 "ID服务器": `op.gaowei.com:21115`
4. 设置 "中继服务器": `op.gaowei.com:21117`
5. 输入公钥
6. 保存设置

### 3. 获取公钥

部署完成后，在服务器上执行以下命令获取公钥:
```bash
cat /opt/rustdesk/data/id_ed25519.pub
```

或者查看部署脚本的输出，公钥会在部署完成时显示。

### 4. 验证配置

1. 重启 Rustdesk 客户端
2. 检查客户端是否显示新的 ID
3. 如果 ID 显示为绿色，说明连接成功
4. 尝试与其他设备建立连接测试

## 常见问题

### Q: 客户端无法连接到服务器
A: 检查以下项目:
- 服务器防火墙是否开放了 21115, 21116, 21117, 21119 端口
- 网络是否能访问 op.gaowei.com
- 公钥是否正确输入
- 服务器服务是否正常运行

### Q: 连接速度慢
A: 
- 检查网络延迟
- 确认中继服务器配置正确
- 考虑使用就近的服务器

### Q: 无法建立P2P连接
A:
- 检查 UDP 端口 21116 是否开放
- 确认 NAT 类型检测正常
- 某些网络环境可能需要通过中继服务器连接

## 高级配置

### 自定义配置文件
可以创建配置文件批量部署:

**Windows**: 创建 `rustdesk.toml` 文件
```toml
[network]
id_server = "op.gaowei.com:21115"
relay_server = "op.gaowei.com:21117"
public_key = "你的公钥内容"
```

## 安全建议

1. **定期更换密钥**: 建议定期重新生成服务器密钥
2. **网络隔离**: 在企业环境中考虑网络隔离
3. **访问控制**: 配置防火墙规则限制访问来源
4. **监控日志**: 定期检查服务器日志

## 故障排除命令

### 服务器端

```bash
# 检查服务状态
docker-compose ps

# 查看日志
docker-compose logs -f

# 检查端口
netstat -tlnp | grep -E ':(21115|21116|21117|21119)'

# 重启服务
docker-compose restart
```
