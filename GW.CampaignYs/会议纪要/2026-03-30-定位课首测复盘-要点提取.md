## 三、产品/流程优化待办（需跟进落地）

### 课评设计优化
- [ ] 横轴拉条改为其他形式（有学员误操作打了1分）
- [ ] 课评问题站在"利他"角度重新设计，拿到更真实反馈
- [ ] 课评分两部分：展示用的好看分数 + 运营改善用的真实反馈

### 诊断报告优化
- [ ] 第一趴问题加入品类信息（如"品牌全称及品类是什么"）
- [ ] 去掉"立即弃用""立即启用"等激进用语，改为"建议"
- [ ] "未调查"项给出具体调查方法建议（如去大众点评/美团/高德买数据）
- [ ] 报告评级用语斟酌（"危险"→"警惕"，考虑更温和的表达）

### 问答环节优化
- [ ] 问答打散为少量多次（3-4次），穿插在课程章节之间，避免疲劳
- [ ] AI辅助处理提问：短的扩写、长的精简、标准化格式
- [ ] 增加必填项（品牌、产品），避免信息缺失导致东哥理解错误
- [ ] 问答名额要**稀缺+精彩**（不是普惠），凸显东哥点评价值
- [ ] 考虑小组内先相互诊断，推一个代表出来让东哥深度点评
- [ ] 第二天提问时间从课程结束后（5点）提前到茶歇后

### 技术/流程优化
- [ ] 多码合一（4→1）需更多测试时间，下次提前准备
- [ ] 复训学员处理流程需提前与运营对齐（这次临时加了十几个账号）
- [ ] 结课流程优化：课评填写与报告下载的顺序问题
- [ ] "润一下"功能鼓励使用（降低AI幻觉率，结构化输入）
