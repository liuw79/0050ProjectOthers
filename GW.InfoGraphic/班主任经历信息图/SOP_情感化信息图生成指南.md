# SOP：情感化信息图生成指南

本指南旨在沉淀本次项目经验，帮助未来更高效、更具情感深度地生成类似的人物经历信息图。

## 1. 核心理念：Data Storytelling (数据叙事)

不要只展示“冰冷的数据”，要展示“数据背后的温度”。
*   **❌ 错误**：他上了 500 节课。
*   **✅ 正确**：他度过了 500 个与学员并肩成长的日夜。

## 2. 标准作业流程 (SOP)

### 第一阶段：情感定调与素材挖掘 (Empathy & Discovery)
在写一行代码之前，先确定“调性”。

1.  **确定主题词**：本次为“拾光”、“见证”、“守望”。
2.  **寻找情感锚点**：
    *   **时间**：第一天是什么时候？当天的天气/心情？（如：2016年那个微冷的冬天）
    *   **人物**：第一次见到重要合作伙伴是什么时候？
    *   **里程碑**：第1次、第100次、第1000次。
3.  **准备素材**：
    *   **核心数据**：CSV/Excel（开课记录、人员名单）。
    *   **感性素材**：照片、语录、喜欢的歌曲（如许巍《家》）、标志性事件。

### 第二阶段：数据结构化与脚本编写 (Scripting)
使用 Python 进行数据清洗，提取“故事线”。

1.  **基础统计**：总天数、总场次、总人数。
2.  **挖掘“第一次”**：
    *   `new_course_milestones`: 每一门课的第1期（代表创新与新生）。
    *   `teacher_first_met`: 与每一位老师的初次合作（代表连接与友谊）。
3.  **挖掘“整数关”**：
    *   `term_milestones`: 第10期、50期、100期（代表坚持与成就）。
4.  **生成 JSON**：将清洗后的数据导出为 `data.json`，供前端调用，实现数据与视图分离。

### 第三阶段：可视化设计与原型 (Prototyping)
使用 HTML/CSS/JS 快速构建响应式长图。

1.  **叙事结构设计**：
    *   **序章 (Prologue)**：散文诗开场，代入情境。
    *   **时光轴 (Timeline)**：按年份推进，穿插大事记（不仅是数据，更是记忆）。
    *   **高光时刻 (Highlights)**：
        *   “见证新生”：用多彩胶囊展示新课诞生。
        *   “里程碑”：用徽章卡片展示重要节点（注意色彩分级）。
        *   “师友同行”：用朋友圈列表展示人际连接。
    *   **总结 (Summary)**：最后才放汇总大数字，作为情感的升华。
2.  **视觉风格 (Visual Identity)**：
    *   **配色**：主色（深邃蓝/高级灰）+ 强调色（暖金/珊瑚红）+ 辅助色（莫兰迪色系）。
    *   **字体**：标题使用衬线体（宋体/楷体）增加人文气息，正文使用无衬线体。
3.  **情感增强 (Emotional Boost)**：
    *   **BGM**：添加符合人物性格的背景音乐。
    *   **金句**：在首屏或尾屏加入触动人心的引言。

### 第四阶段：打磨与交付 (Polishing)
1.  **细节微调**：
    *   检查颜色语义（如：50期不要用灰色，要用热烈的红或高级的银）。
    *   检查文案温度（把“数据统计”改为“硕果·份量”）。
2.  **多端适配**：确保手机端浏览体验最佳（通常是长图模式）。

## 3. 关键代码模式 (Code Patterns)

*   **色彩哈希**：为大量标签自动生成不重复的柔和背景色。
    ```javascript
    const colorIndex = Math.abs(hash) % palette.length;
    ```
*   **时间轴渲染**：支持多种事件类型（summary, personal, milestone）的混合渲染。

## 4. 用户偏好记录 (User Preferences)
*   **风格**：偏好简洁、有艺术感、有温度的设计，不喜欢枯燥的列表。
*   **音乐**：喜欢许巍等有人文情怀的歌曲。
*   **色彩**：喜欢暖色调或有质感的颜色，避免消极色（如灰色）。
