# GW.Content 2.0 设计文档

## 一、设计背景

### 旧系统问题
- 人的操作度太低，只是跑流程
- 喂给 AI 的内容不透明、不清晰
- 用户不知道 AI 为什么这样写

### 新系统目标
**"素材透明，流程可控"** — 用户全程看到 AI 吃什么、做什么

---

## 二、核心需求

| 维度 | 决策 |
|------|------|
| 控制度 | 全程可控（选材、配置、预览、调整） |
| 使用场景 | 小团队协作（2-3人），区分私有/公共素材 |
| 交互模式 | 工作台模式，分步引导 |
| 素材组织 | 课程素材 + 通用素材（用户画像/品牌/参考标题） |
| 素材来源 | AI 从课程原文提取，用户审核确认 |
| 数据存储 | 飞书多维表格 |

---

## 四、用户流程

### Step 1: 确定选题

**两种输入方式：**

1. **选课程** — 用户选择一门或多门课程
2. **给关键词** — 用户输入关键词或选题想法

**AI 返回：**
- 3-5 个推荐选题
- 一个开放输入框让用户自己填写

**用户操作：**
- 从推荐中选择一个，或
- 自己填写标题
- 点击"确认选题"进入下一步

---

### Step 2: 素材准备

**AI 自动准备多维度素材：**

1. **课程素材**（来自选定课程）
   - 显示标题 + 内容预览（可展开看全文）
   - 默认勾选相关素材
   - 用户可调整勾选

2. **用户画像**
   - 目标用户特征
   - 核心痛点

3. **高维品牌**
   - 高维是谁
   - 语言风格
   - 五不学习约定

4. **参考标题**
   - 爆款标题风格示例

**用户操作：**
- 预览每条素材内容
- 调整勾选（增/删）
- 点击"预览完整 Prompt"查看 AI 将收到的所有内容
- 确认后点击"开始写作"

---

### Step 3: 生成内容

- AI 根据选题 + 素材 + 配置生成文章
- 显示生成结果
- 用户可：
  - 调整素材/配置后重新生成
  - 保存草稿
  - 导出/发布

## 五、页面结构

### 用户端

```
├── 首页（写作项目列表）
├── 新建项目
│   ├── Step 1: 确定选题
│   ├── Step 2: 素材准备
│   └── Step 3: 生成内容
└── 我的（个人设置）
```

### 管理端（后台）

```
├── 课程管理
│   ├── 课程列表
│   ├── 上传原文
│   └── AI 提取素材
├── 素材管理
│   ├── 素材列表
│   ├── 新建/编辑素材
│   └── 审核素材
└── 用户管理
```

## 六、技术架构

### 前端
- React / Vue（待定）

## 七、与旧系统的主要变化

| 方面 | 旧系统 | 新系统 |
|------|--------|--------|
| 素材选择 | 用户手动勾选 | AI 推荐 + 用户调整 |
| 流程可见性 | 黑盒 | 分步引导，每步可见 |
| Prompt 透明度 | 不可见 | 可预览完整 Prompt |
| 素材维度 | 仅课程素材 | 课程 + 用户画像 + 品牌 + 参考标题 |
| 素材预览 | 仅标题 | 标题 + 内容预览 |
