# 写作工作台设计文档

**日期**: 2026-02-14
**状态**: 已批准

---

## 1. 概述

为 GW.Content 系统添加公众号写作工作台，实现完整的 9 步写作流程。

### 核心功能
- 选题建议（自动生成/关键词扩展/评估）
- 大纲生成与编辑
- AI 辅助写作（分段/全文）
- 三遍审校降 AI 味
- 发布到飞书文档
- Skill/素材库迭代建议

---

## 2. 技术方案

### 2.1 架构

**单页面工作流** - 一个页面完成全部写作流程，通过步骤切换。

### 2.2 前端

- `writing-workbench.html` - 写作工作台主页面
- 复用 `common.css` 和 `common.js`

### 2.3 后端 API

```
POST /api/topics/suggest      # 根据素材生成选题建议
POST /api/topics/evaluate     # 评估选题优劣势
POST /api/outlines/generate   # 生成大纲
POST /api/articles/write      # AI 写作（分段/全文）
POST /api/articles/review     # 审校降 AI 味
POST /api/articles/publish    # 发布到飞书文档
POST /api/suggestions/generate # 生成迭代建议
POST /api/suggestions/adopt   # 一键采纳建议
```

### 2.4 飞书文档集成

扩展 `lark-client.js`，添加：
- `createDocument(title, content)` - 创建文档
- `getDocument(documentId)` - 获取文档内容

## 3. UI 设计

### 3.1 页面布局

```
┌─────────────────────────────────────────────────────┐
│  写作工作台                    [素材库] [文章列表]    │
├─────────────────────────────────────────────────────┤
│  步骤指示器: [1.选题]─[2.大纲]─[3.写作]─[4.审校]─[5.发布] │
├──────────────┬──────────────────────────────────────┤
│   左侧面板    │           主编辑区                   │
│  (固定宽度)   │         (自适应宽度)                 │
├──────────────┴──────────────────────────────────────┤
│  [上一步]                      [AI辅助]    [下一步]  │
└─────────────────────────────────────────────────────┘
```

### 3.2 各步骤内容

| 步骤 | 左侧面板 | 主编辑区 |
|------|----------|----------|
| 1.选题 | 素材列表 + 关键词输入 | 选题建议卡片（3-4个） |
| 2.大纲 | 选题摘要 | 大纲编辑器（可拖拽排序） |
| 3.写作 | 大纲结构 | Markdown 编辑器 |
| 4.审校 | AI 审校建议 | 文章预览 + 修改对比 |
| 5.发布 | 发布设置 | 飞书文档预览 + 迭代建议 |

## 4. 流程设计

### Step 1: 选题
1. 显示素材库列表（可搜索筛选）
2. 勾选素材 → 生成选题建议
3. 每个选题显示：标题、角度、难度、优劣势
4. 支持：关键词扩展、自定义选题评估

### Step 2: 大纲
1. 显示选题摘要
2. AI 生成 5-7 章节
3. 支持：拖拽排序、增删章节、设置字数

### Step 3: 写作
1. 两种模式：分段写作 / 全文生成
2. Markdown 编辑器
3. 实时预览

### Step 4: 审校
1. 三遍审校：内容 → 风格 → 细节
2. 显示问题列表
3. 一键采纳或手动修改
4. 显示 AI 检测分数

### Step 5: 发布
1. 设置标题、文件夹
2. 发布到飞书文档
3. 生成迭代建议
4. 一键采纳建议
