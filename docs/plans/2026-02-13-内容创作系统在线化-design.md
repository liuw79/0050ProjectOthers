## 1. 项目概述

### 1.1 项目背景

当前是一个基于本地文件系统和 Claude Code CLI 的内容生产系统，通过文档驱动的工作流程来创作文章、主持词、朋友圈文案等内容。

目标是将其改造为在线系统，供4-10人团队独立使用，统一管理内容和素材。

### 1.2 核心目的

1. 让团队成员能够独立使用AI创作内容，无需管理员介入
2. 构建统一的内容管理和素材管理平台

### 1.3 核心任务

不是简单的"让AI写文章"，而是一套 **AI辅助的内容创作方法论和工作流**：

- 9步完整流程（理解需求→选题→学习风格→用素材库→创作→三遍审校→配图）
- 两层判断机制（先判断工作区，再判断任务类型）
- 核心协作原则（Think Aloud透明化思考、调研先行）
- **降低AI味** - 通过个人素材库、风格学习、三遍审校

### 1.4 关键点

1. **降低AI味** - 核心特色，通过个人素材库、三遍审校、学习用户风格
2. **工作流引擎** - AI需要理解并执行复杂的多步骤流程
3. **知识管理** - 课程内容、个人素材库、写作参考的结构化管理
4. **透明化协作** - AI说明思考过程，用户可干预关键决策
5. **知识点拆分** - 课程内容需要被结构化拆解成可复用的知识单元
6. **Skill迭代闭环** - 生成→AI评估→人工确认→分析→优化的持续改进

### 1.5 包含范围

✅ **包含：**
- 内容创作工作流（公众号、主持词、朋友圈、五句话介绍等）
- 个人素材库管理
- 写作参考/风格指南管理
- 课程全文管理（仅AI调用，不展示给用户）
- 已拆好的知识点管理（标签、分类、关联）
- Skill自我迭代闭环（生成→评估→分析→优化）
- 多模型支持（Gemini提取知识、Claude迭代skill等）

❌ **不包含：**
- 知识点的拆分过程（线下或其他方式完成）
- 团队协作功能（暂时不需要）

## 3. 技术方案选择

### 3.1 方案对比

| 方案 | 技术栈 | 优点 | 缺点 |
|------|--------|------|------|
| 飞书应用 | Vue3 + 飞书多维表格 + 飞书文档 | 完全国内环境，访问快；权限控制原生支持；维护成本低 | 受限于飞书平台，定制能力有限 |
| Next.js + Supabase | Next.js + Supabase | 开发效率高；Supabase提供开箱即用的数据库、认证、权限 | 国内访问可能较慢 |
| 自建 | Vue3 + 自建后端 + 阿里云RDS | 完全自主可控 | 开发复杂度高，维护成本高 |

### 3.2 选择方案：飞书应用

**选择理由：**
1. 用户在国内，飞书访问速度快、稳定
2. 飞书多维表格原生支持权限控制，适合课程内容权限隔离需求
3. 飞书文档支持长文本存储，适合课程全文
4. 维护成本极低
5. 团队可能已在用飞书，学习成本低

### 6.2 内容创作流程

**流程进度条展示：**
```
步骤1/9：理解需求 → ★当前步骤 → 步骤3：选题讨论 → ... → 步骤9：完成
```

**每个步骤的交互方式：**
- 系统展示：当前步骤的说明、AI的思考过程（Think Aloud）
- 用户操作：输入信息/选择选项/确认下一步
- 等待AI处理：显示加载动画

**关键交互节点：**
| 步骤 | 用户需要做什么 |
|------|----------------|
| 步骤1 | 输入或确认需求 |
| 步骤3 | 选择选题方向 |
| 步骤6 | 确认使用哪些素材（可选） |
| 步骤8 | 三遍审校后，确认内容 |
| 步骤9 | 查看最终内容，保存/编辑 |

## 7. 工作流引擎设计

### 7.1 核心概念
将当前的文档驱动工作流转化为代码驱动的工作流引擎。

### 7.2 工作流定义

每个内容类型对应一个工作流，由多个步骤组成：

```javascript
// 公众号写作工作流定义
const publicAccountWorkflow = {
  steps: [
    {
      id: 'step1',
      name: '理解需求',
      type: 'user_input',
      description: '请输入你的写作需求',
      fields: ['title', 'target_audience', 'length', 'key_points']
    },
    {
      id: 'step2',
      name: '信息搜索',
      type: 'ai_process',
      model: 'gemini-2.0-flash', // Gemini负责知识提取
      description: 'AI正在搜索相关信息...'
    },
    {
      id: 'step3',
      name: '选题讨论',
      type: 'ai_generate_and_user_select',
      model: 'claude-sonnet-4.5',
      description: 'AI正在生成选题方向...',
      selectOptions: 3-4
    },
    // ... 其他步骤
  ]
}
```

### 7.3 模型调度策略

**模型特点：**
| 模型 | 擅长领域 |
|------|---------|
| Gemini Flash | 快速搜索、信息提取 |
| Gemini 3.0 Pro | 内容生成、知识提取 |
| Claude Sonnet | 内容评估、写作 |
| Claude Opus | 技能迭代、复杂分析 |
| GLM-5 | 中文内容生成、长文档处理、中文语境理解 |

| 步骤类型 | 默认模型 | 可选模型 |
|---------|---------|---------|
| 知识提取 | Gemini Flash | Gemini Pro / GLM-5 |
| 内容生成 | GLM-5 / Gemini 3.0 Pro | Claude Sonnet/Opus |
| 内容评估 | Claude Sonnet | GLM-5 / Gemini 3.0 Pro |
| 技能迭代 | Claude Opus | GLM-5 / Gemini 3.0 Pro |

### 7.4 AI调用层设计

```javascript
// AI调用层接口
class AIClient {
  async callClaude(prompt, model, options) {
    // 调用Anthropic API
  }

  async callGemini(prompt, model, options) {
    // 调用Google AI API
  }

  async callGLM(prompt, model, options) {
    // 调用智谱AI GLM API
  }

  async callModel(taskType, prompt, options) {
    // 根据任务类型自动选择模型
    const model = this.selectModel(taskType);
    if (model.startsWith('claude')) {
      return this.callClaude(prompt, model, options);
    } else if (model.startsWith('gemini')) {
      return this.callGemini(prompt, model, options);
    } else if (model.startsWith('glm')) {
      return this.callGLM(prompt, model, options);
    }
  }
}
```

## 8. Skill迭代闭环设计

### 8.1 迭代流程图

```
┌─────────────────────────────────────────────────────────┐
│                   Skill迭代闭环                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  现有Skill (v2.3)                                      │
│       ↓                                                │
│  生成内容 (15篇)                                        │
│       ↓                                                │
│  ┌─────────────────────────────────┐                   │
│  │ AI评估                          │                   │
│  │ - 自动评分                       │                   │
│  │ - 生成评估反馈                   │                   │
│  └─────────────────────────────────┘                   │
│       ↓                                                │
│  ┌─────────────────────────────────┐                   │
│  │ 人工确认                        │                   │
│  │ - 查看AI评分和反馈              │                   │
│  │ - 人工评分 (好/一般/不好)       │                   │
│  │ - 添加人工反馈                  │                   │
│  └─────────────────────────────────┘                   │
│       ↓                                                │
│  数据分析 (查看评分趋势)                               │
│       ↓                                                │
│  如果平均评分下降或反馈不好？                           │
│       ↓                                                │
│  ┌─────────────────────────────────┐                   │
│  │ 分析原因                        │                   │
│  │ - 哪些内容评分低？              │                   │
│  │ - 用户反馈集中在什么问题？       │                   │
│  │ - 可能的改进方向？              │                   │
│  └─────────────────────────────────┘                   │
│       ↓                                                │
│  ┌─────────────────────────────────┐                   │
│  │ AI辅助迭代                      │                   │
│  │ - 输入：当前Skill + 分析结果    │                   │
│  │ - 输出：新Skill版本 (v2.4)      │                   │
│  └─────────────────────────────────┘                   │
│       ↓                                                │
│  人工确认/调整                                         │
│       ↓                                                │
│  新Skill (v2.4) → 回到起点                              │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 8.2 迭代触发条件

**自动触发：**
- 连续5篇内容评分低于阈值（如3.0/5）
- 平均评分相比上版本下降明显

**手动触发：**
- 管理员主动发起迭代
- 用户在反馈中提出明确问题

### 8.3 迭代记录

每次迭代保存：
- 版本号（v2.3 → v2.4）
- 迭代原因（评分下降/用户反馈/主动优化）
- 改进内容（增加了什么/减少了什么）
- 创建时间
- 对比数据（迭代前后的平均评分）
