## 产品规划：CEO人才雷达（暂定名）

### 1. 产品名称
*   **建议名称：** CEO人才雷达 / 伯乐纪 / 将才录
*   **说明：** 名称强调CEO视角、关键人才、以及持续追踪的特性。

### 2. 产品定位
一款专为创业企业CEO及高管设计的**轻量级关键人才追踪与互动促进工具**，通过智能化提醒与便捷记录，提升“找人”而非“招人”的效率与效果，构建动态的核心人才池。

### 3. 目标用户
*   创业公司CEO、创始人
*   公司高层管理人员（如VP、CXO等）

### 4. 核心痛点
*   **信息分散难管理：** 优秀人才信息散落在微信、邮件、笔记等各处，难以统一管理和追踪。
*   **互动不及时易遗忘：** 日常繁忙，容易忘记与关键人才保持联系，错失良机。
*   **缺乏系统性跟进：** 对人才的跟进缺乏计划性和有效方法，互动效果不佳。
*   **团队找人不成体系：** 高管团队各自为战，人才信息和进展不透明，难以形成合力。
*   **时间精力投入衡量难：** 无法有效评估在“找人”上投入的时间和精力。

### 5. 核心功能设想

#### 5.1. 人才录入与管理
*   **快速录入：** 支持手动输入、名片扫描（OCR可选）、从微信等渠道快捷导入（技术可行性待调研）人才基本信息（姓名、联系方式、来源、备注等）。
*   **语音备注：** 方便CEO快速记录与人才相关的想法、沟通要点。
*   **标签化管理：** 可自定义或预设标签（如：价值观、专业能力、创业精神、领导力、创新、成功项目经历等）对人才进行分类。
*   **人才池构建：** 形成公司专属的动态核心人才数据库。

#### 5.2. 智能追踪与互动提醒
*   **AI定时提醒：** 根据设定的周期或人才重要性，智能提醒CEO/高管与特定人才进行互动。
*   **互动记录：** 记录与人才的每次互动（如发送问候、约咖啡、分享公司进展等）。
*   **预设跟进方式：** 提供多种跟进方式模板（如：发送公司最新进展、节日问候、邀请参加活动、赠送小礼物、产品体验邀请等），方便快速发起互动。
*   **互动频率建议：** AI根据人才重要性、上次互动时间等因素，建议合适的互动时机和方式。

#### 5.3. 人才评估与决策辅助
*   **简易评估模型：**
    *   沟通后引导用户回答关键问题：
        1.  能帮你实现战略目标吗？（是/否/继续考察）
        2.  能提升公司这方面能力吗？（是/否/继续考察）
        3.  比你（或团队现有成员）水平高吗？（是/否/继续考察）
    *   记录评估结果，辅助判断。
*   **“关键损失”标记：** 对于特别重要的人才，可标记“若不加入，损失重大”。

#### 5.4. 团队协作与信息共享
*   **高管共享人才池：** 团队内高管可共同维护和查看人才池信息（权限可设）。
*   **人才进展互通：** 团队成员可了解彼此与人才的互动进展，避免重复打扰或协同跟进。
*   **“人才PK会”辅助：** 开会时可快速调取人才信息，方便团队讨论和评估。
*   **HR后台视角（可选）：** HR可查看整体人才库情况及高管互动频率，用于激励或提供支持。

#### 5.5. AI辅助功能
*   **人才池智能搜索：** 根据关键词、标签、备注等快速在人才池中搜索人才。
*   **语音转文字：** 将语音备注自动转换为文字，方便查阅和搜索。
*   **个性化提醒语：** AI根据不同场景和人才特点，生成多样化的提醒内容。

### 6. MVP (最小可行产品) 功能列表

为了实现“极轻”和快速验证核心价值，MVP应聚焦于以下功能：

1.  **核心人才信息录入：**
    *   手动输入姓名、联系方式、来源渠道、初步备注。
    *   支持简单的标签（如：非常重要、重点跟进、一般关注）。
2.  **手动设置互动提醒：**
    *   用户可以为每个人才设置下一次需要互动的日期和简单提醒事项。
    *   到期后小程序推送提醒。
3.  **互动记录：**
    *   简单记录与人才的互动类型（如：微信沟通、电话、约见）和日期、简要内容。
4.  **基础人才列表与查看：**
    *   按添加时间、提醒时间等简单排序和筛选。
5.  **单用户使用：** MVP阶段首先满足CEO个人使用场景。

**MVP目标：** 验证CEO是否愿意使用这样一款工具来记录和提醒自己跟进关键人才，并初步感受其便利性。

### 7. 后续迭代方向
*   **AI智能提醒优化：** 基于互动历史和人才画像，提供更精准的提醒时机和内容建议。
*   **团队协作功能完善：** 逐步开放高管团队共享、评论、协同跟进等功能。
*   **人才评估体系深化：** 引入更结构化的评估维度和打分机制。
*   **AI智能搜索与匹配：** 根据公司需求和岗位画像，在人才池中智能推荐合适人选。
*   **数据分析与洞察：** 分析人才互动数据，为CEO提供关于人才获取效率、时间投入等方面的洞察。
*   **外部渠道对接（可选）：** 探索与招聘平台、社交媒体等渠道的轻度整合。
*   **AI面试陪练/辅助（远期）：** 结合您提到的“AI面试陪练体”，在后期可以考虑引入，帮助CEO和高管提升面试技能，但这部分功能较重，不建议早期投入。
*   **语音交互深化：** 更智能的语音录入、查询和互动。

### 8. 技术选型建议 (小程序)
*   **前端：** 微信小程序原生开发或Taro/uni-app等多端框架（考虑“极轻”，原生可能更优）。
*   **后端：** Node.js + Serverless (如云开发) / Python + Flask/Django / Go。云开发可以快速启动，降低初期运维成本。
*   **数据库：** MongoDB / PostgreSQL / MySQL (根据具体需求选择，云开发自带数据库)。
*   **AI能力：** 早期可依赖第三方成熟的NLP API（如语音识别、文本分析），后期若有更复杂需求可考虑自建模型或定制服务。

这份规划希望能为您提供一个清晰的起点。核心在于快速上线MVP，收集用户反馈，然后小步快跑，持续迭代。
