# STEP 1：选集成操盘模型、定操盘手、建战队

## 📊 概述

STEP 1是操盘战队集成训战七步法的第一步，重点解决"哪些人一起协作构成核心作战团队"的问题。通过选择合适的操盘模型、确定操盘手、建立作战团队，为后续的集成作战奠定组织基础。

## 🏗️ 不同操盘阶段的商业模式特征

- **操盘1.0阶段**: 营销相对粗放生长
- **操盘2.0阶段**: 多渠道规模突破
- **操盘3.0阶段**: 精细化、全渠道规模破局
- **操盘4.0阶段**: 深度裂变，再到质变

## 🎯 集成操盘管理--操盘团队顶层设计

### 模型1：1层操盘 (操盘1.0阶段 - 营销相对粗放生长)
- **特征**:
  - 企业产品结构简单，品类单一
  - CEO/高层决策者为项目组成员，直接操盘并决策
- **层级**:
  - **直接操盘决策**

### 模型2：2层操盘 (操盘2.0阶段 - 多渠道规模突破)
- **特征**:
  - 企业产品结构简单，奶牛品类相对单一
  - 实现多渠道突破
  - 有孵化新品类、新市场，但不复杂
  - 分层分级授权
- **层级**:
  - **前端利润中心直接操盘**
  - **高层看护操盘、战略操盘**

### 模型3：3层操盘 (操盘3.0阶段 - 精细化、全渠道规模破局)
- **特征**:
  - 企业规模大，奶牛品类多个，主力市场多个
  - 有孵化新品类、新市场，操盘复杂
  - 分层分级授权
- **层级**:
  - **前端利润中心直接操盘 (国家/品类)**
  - **营销服/战区VP看护操盘**
  - **高层战略操盘**

## 💭 思考与Tips

### 思考题
- 我们现在所处在哪个操盘阶段？
- 我们现在是哪种操盘团队模式？
- 我们应该是哪种操盘团队模式？
  - 不同品类与不同市场的考虑

### Tips
1.  **企业高层要承担以经营为导向的关键决策责任**
2.  **分层分级授权、决策是企业未来思考的方向-善管理，建立会议机制**
    -   激发组织活力：从"要我做"到"我要做"
    -   科学的评价与激励机制：授权必须与责任和利益对等
    -   赋能与支持，而非放任自流
3.  **"先人后事"。一号位画像、照镜子、个人成长计划**
    -   敢于自我剖析，共成长，为公司创造价值

## 🧠 认知升级才能破局

- **产品的成功是市场成功的基础**
  - 即便品牌、营销、渠道、用户、资金实力等要素有深厚积累，产品不能持续成功也会被市场淘汰。(以Nokia, Motorola为例)
- **市场的成功依托于领先的集成作战的流程**
  - 从产品为投资主线经营导向，责任共担各有侧重，转变为从分散作战到集成作战。

## 🎯 目标落地需要力出一孔的强力战队，明确的商业成功责任人

### 破局方案=胜利——没有铁军，再锋利的刀也会卷刃

**集成是胜利的关键要素**

- 根据McKinsey Global Survey, 2018的调查数据，导致战略失败的主要原因包括：
  - 部门目标冲突与协同断裂
  - 战略方案本身缺陷
  - 外部环境突变
- "绝大多数战略溃败并非源于糟糕的规划，而是组织无法围绕目标形成合力。当市场团队抱怨'产品交付太慢'、产品团队指责'销售扭曲需求'时，再完美的方案也将沦为废纸。"

### 本质是回答：哪些人一起协作构成核心作战团队？

**集成营销操盘注重解答三个问题**:
- **What**: 产品目标是什么？
- **Who**: 哪些人一起团队协作？
- **How**: 如何实现目标？

## 🔄 打破原有运行模型组建操盘的作战团队

### 职能型组织组阁的问题
1.  **串行**: 上市交付提环节节点多，易产生"延迟效应"
2.  **部门墙**: 职责频繁转移、信息层层衰减，易滋生"本位主义"

### 跨部门团队带来的变化
1.  **目标一致**: 团队关注于产品，为产品的成功负责。
2.  **并行、协同**: 各环节活动更充分，协作效率高，容易构建组织能力。
3.  **一个团队**: 打破部门界限组建混合团队，端到端负责，信息始终共享。
4.  **决策均衡、全面**: 代表多方利益博弈制衡，多领域知识经验集成。

### 跨部门团队的典型特征
1.  项目负责人**专职化**，对项目目标负责。
2.  项目成员被**充分授权**，可以代表各功能领域参与项目决策和做出承诺。
3.  项目负责人对成员有**一定的考核权**。
4.  项目负责人管理项目预算，有**预算范围内决策的决策权**。

## 👥 以PCT Leader为核心的集成作战团队

### PCT集成作战团队构成
- **核心圈 (PCT Leader)**: 经营、4P、项目管理
- **内圈 (核心兵种)**:
  - **GTM代表**: 货、价、量、项目管理
  - **MKT代表**: 价值主张/品牌
  - **渠道代表**: 渠道操盘
  - **零售代表**: 物料及标准、终端沟通
- **外圈 (外围角色)**:
  - 定价商务代表
  - 市场计划代表
  - 服务代表
  - 财经代表
  - 交付代表

### PCT团队的运作
- 组建由PCT Leader领导的PCT (Product Commercial Team) 项目组推动GTM营销操盘管理工作，确保IPD/IPMS的执行互锁。
- 除四个核心兵种外，PCT团队在工作中如遇其他需协同事项，则按需卷入其他领域代表，参与PCT会议。
- PCT团队既有**核心**也有**外围角色**。

## 📋 PCT集成作战团队的职责

### PCT负责什么？
是IPMS流程中市场体系下跨功能部门的产品市场操盘团队，负责产品在市场操盘整个过程中的端到端管理(确保产品卖得好)

### PCT的目标是什么？
保证产品在市场上取得商业成功(规模和利润)。对产品的整体操盘(价、量、利润)负责，包括产品操盘(进度、收入、客户满意)和按时保量完成销售

### PCT的主要工作是什么？
1. 管理和执行产品操盘过程中各种不同的业务和市场要素及时做出决策，保证各领域活动和各阶段交付件按时按质完成；
2. 产品上市过程中的评审关键评审点(如产品各阶段GR点)；
3. 拉通产品上市过程中及上市后的GTM、营销、渠道、零售、服务等各领域进度及各领域的信息共享；
4. 对产品生命周期内量价的波动进行监控并执行IPMS 操盘会的决策，保证产品的最大收益；
5. 对PCT各领域的重大变更进行评估(拓展策略变更、进度变更、街价变更、范围变更等)提交IPMS 操盘会决策；
6. 输出GR的结论作为市场领域的操盘建议给PDT，由PDT提交IPMT，支撑投资决策。

## 🔗 PCT Leader & GTM的关系

| PCT Leader (操盘手角色) | GTM (核心兵种角色) |
| :--- | :--- |
| **商业结果owner (市场/产品)** | 项目管理责任人、操盘/复盘/退市提案人 |
| - 操盘方案制定与决策 | - 产品量价策、预算管理提案 |
| - 关键客户关系 | - 机会分析、竞争分析 |
| - 识别机会、风险, 关键事项闭环 | - 项目管理, 组织关键会议、节点评审 |

- **小规模利润中心**，可以考虑GTM由PCT leader或者其他核心代表兼任
- **大规模利润中心**，可以考虑设置专职GTM，授予一定授权

## 📝 总结

STEP 1的核心在于建立正确的组织架构和团队配置。通过选择合适的操盘模型、确定PCT Leader、组建跨部门作战团队，为后续的集成作战奠定坚实的组织基础。关键在于打破部门墙，实现目标一致、并行协同的作战模式。
