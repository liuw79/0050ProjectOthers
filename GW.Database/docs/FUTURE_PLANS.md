# GW.Database - 未来计划

本项目旨在提供一个基础的数据库结构查看器。为了使其更加强大和实用，我们有以下未来计划和潜在的改进方向。

## 1. 数据字典整合与优化

-   **结构化数据字典**: 将现有分散、混乱的数据字典文档（例如 Excel、Word、Markdown 等）解析并整合为统一的结构化格式（如 JSON 或 YAML）。
-   **API 增强**: 扩展 `/tables/{table_name}` API，使其能够融合数据库内省结果和数据字典中的人工描述信息，提供更全面的表和字段视图。
-   **数据字典管理界面**: 开发一个简单的 Web 界面，允许用户直接在浏览器中编辑和维护数据字典，而无需手动修改文件。

## 2. 真实数据查询功能

-   **数据查询 API**: 增加新的 API 端点（例如 `/data/{table_name}`），允许用户查询指定表的部分真实数据，以验证数据结构和内容。
-   **查询参数**: 支持分页、过滤和排序等查询参数，以满足更复杂的数据探索需求。
-   **数据导出**: 提供将查询结果导出为 CSV 或 Excel 格式的功能。

## 3. 用户界面 (UI) 改进

-   **美观的 Web 界面**: 基于 FastAPI 的前端框架（如 Jinja2 模板引擎结合 Bootstrap/Tailwind CSS），构建一个更直观、用户友好的 Web 界面，替代默认的 Swagger UI。
-   **搜索和过滤**: 在 UI 中实现对表和字段的快速搜索和过滤功能。
-   **关系图谱**: 尝试生成简单的数据库表关系图（ERD），帮助用户理解表之间的关联。

## 4. 数据库支持扩展

-   **多数据库支持**: 扩展对其他数据库类型（如 MySQL, PostgreSQL, SQLite）的支持，使其成为一个通用的数据库查看工具。

## 5. 部署与容器化

-   **Docker 支持**: 提供 Dockerfile 和 Docker Compose 配置，简化项目的部署和管理。
-   **生产环境部署指南**: 提供在生产环境中部署 FastAPI 应用的详细指南（如使用 Gunicorn + Nginx）。

## 6. 性能优化与安全性

-   **连接池优化**: 针对高并发场景，优化数据库连接池的配置。
-   **认证与授权**: 为 API 添加认证和授权机制，保护敏感数据。
-   **日志与监控**: 完善日志记录，并集成监控工具。

## 贡献

我们欢迎社区成员对以上任何一个方向进行贡献。如果您对某个功能感兴趣，或者有新的想法，请随时提出。
